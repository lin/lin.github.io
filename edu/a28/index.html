Learning as a Rancher

  1. 简单，努力就有进展，目标明确，不用太思考

  2. 软性指引，不强迫，但是由于游戏性，而自发的配置资源

  3. 大量练习，从而扎实掌握基本功，得到成就感

  4. 房间的路径设置

  5. 传送点如何设置

  6. 如何让学生能软性复习之前的错题（非主动去做）

什么时候？学生能自主选择做什么样的题目？

假设：数量和质量有一定的正相关，让数量成为目标，成为质量的一个近似

  1. 做事就要有积累，有进展，关键的量积累，合成素材，从而体现进展，也就是通过数量的指引，达到质量的改变
  2. 做题就要有收益，让努力得到收益
  3. 宠物要积累（宠物意味着努力的程度，也意味着学生只要努力就有收益）
  4. 牧场要升级（牧场意味着技能的等级，尽可能的和技能实际水平相关，也就是说这块的技能稳定了，不容易崩溃了）

牧场

  1. 每个宠物都有自己的牧场，牧场的级别对应着技能的等级
  2. 牧场可以有很多的宠物，宠物越多，产出越多；产出越多，收益越高，级别也就越高
  3. 需要花钱来提升牧场的等级，优化牧场的自动化，升级牧场，增加便利
  4. 钱可以从销售宠物的产物获得，意味着学生的努力多
  5. 更难的题目，使得升级牧场的速度更快（钱多或者素材多），因为这意味着技能的高等级
  6. 每个牧场有收益上限，也就是不能无限放置

野外

  1. 做题就是出去采摘和打猎
  2. 打猎需要respawn的时间，采摘也是，cd时间更短
  3. 野外有传送门，如果学生等级高，有机会直接去挑战高难度的区域
  4. 如果在高难度的区域，总是失败，那么需要消耗素材或者cd，才能再去高难度
  5. 当某一区域，积累了足够的宠物种类和级别，就能解锁这个区域（也就是大门）
  6. 区域boss战，可能得到奖杯，也就是定级考试

关键类比

  1. 证书，也就是技能水平，是牧场的等级和收益能力
  2. 金币，也就是经验值，意味着玩家的努力程度（做上高难度的题目也可以迅速积累）

Simple

目标

  1. 简单，计算少，尽可能不用存储
  2. 评价，能够给予强烈的学习目标感
  3. 目标多，方向明确，无脑执行，软性指引
  4. 有新鲜感，不要太无聊

要点

  1. 技能
  2. 玩家对技能的记忆强度，流畅程度
  3. 技能的考察频率，重要程度
  4. 技能的难度，复杂度
  5. 玩家对技能的难度深度

难点或者冲突点

  1. 想让玩家更早接触高级技能（传送门），又希望玩家能通过解锁来奖励（大门打开）
  2. 想让玩家接触到更多的技能（可以解锁区域），又希望玩家能够反复锤炼基本功（捉宠物，养成宠物）
  3. 想让玩家挑战难题（想证明自己），但又希望他们能多做基础题（需要积累足够的证书）
  4. 想通过积累（财富或者人口）来简化驱动（可以捕捉更多的宠物），又希望通过探索和挑战关卡实现（有boss战，以及考证赛）

你可以证明自己，但是难度和级别差距越大，cd时间越长，别总做难题，需要基本功来证明自己。

基本要素

  1. 📜 证书，宝可梦
  2. ✨ 便利，通过财富或者人口解锁
  3. 💯 分数，要么是做上难题（elo），要么是积累量的提高（rpg）

翻译

素材，工人，产品，自动化

  1. 做基础题可以积累解锁的素材，达到一定明确指标之后，可以自动化
  2. 做难题如果失败，需要提供更多的素材，才能再次挑战
  3. 做题，就是收集材料，收集宠物，上野外搞事情
  4. 生产链关系，也就是一个技能或者宠物的食物，是另一个宠物的产物
  5. 不要太一一对应，而是差不多对应，否则会让玩家过于难受
  6. 不要让玩家思考过多，纠结过多，有个差不多的近似模型是最好的
  7. 单一的经济系统，这样玩家可以更加清晰的知道利害得失，以及收益的暴涨，从而左右自己的行为

直接翻译

  1. 重要程度，就是收购价格，可以让学生感到值钱，愿意多刷这些技能的题目
  2. 难度，就是xp奖励高，xp的提高，可以提高lv，lv越高，很多设施成本越低
  3. 等级，就是农场等级，也就是证书可信度的逐步提高
  4. 武器，
  5. 防具，

基本机制

  1. 遇到宠物，抓到宠物，驯化宠物，牧养宠物，带着宠物去战斗
  2. 宠物有依赖关系，需要食物，需要饲养环境，能够产生食物，食物可以去卖
  3. 经济系统就是可以升级，可以建造耕作的饲养环境，但必须是同级别的宠物
  4. 由于知识需要一定的时间消化，那么有些地方想收割，需要cd
  5. 宠物有量，就像slime，就像pal，就像pokemon

路径问题

可扫，可练，可深，可跳

软性指引，技能评价，逐步解锁，收益变化

  1. 怎么能迅速跳出简单的题，显得无聊（但你必须做几个简单题，来证明自己配做难题，打开做难题的大门）
  2. 怎么能反复的挑战上一次难题（但如果你只想做难题，那也不行，需要一定的cd时间来由机会做难题）
  3. 怎么能迅速的过基本的知识点，扫一遍（也就是第一层大门很容易打开，也就是百科很容易解锁）
  4. 怎么能把一个知识练好，刻意练习（每一个大门都有深度，有练习的成分）
  5. 怎么能迅速定位一个知识点（每个区域，或者每个关卡，都是一个）
  6. 怎么能跳到一个更厉害的区域（直升机）

需求分析

让学生自己选择门，是必须的要素

  1. 基础好的，不用再受基本功的折磨，直升机蹿升
  2. 学得好了，也可以练基本功，而不是就没有机会了，只是收益变小了

提供入口和房间，提供自由选择的门，但是通过奖励的大小，来软性指引学生的行为。

也就是说，这里会有大量的进度条，但是学生根据自己的需求来选择推进那一个进度条

  1. 喜欢一次性先过一遍
  2. 喜欢稳扎稳打，一个一个的推进
  3. 喜欢做难题，通过难题来进展
  4. 喜欢做简单题，通过简单题来推进

可以进门，但是要承担代价

必须有的要素

  1. 每个房间结束必须有多个门（三个门，或者第四个），必须有所取舍，各有利弊
  2. 单向门，只有到达了之后，才能打开的双向门（门不能从这侧打开）
  3. 解锁大门，也是一个重要的进展符号（探索的解锁），有种传送点被点开的感觉

时刻的技能评价以及收益与路径变化
